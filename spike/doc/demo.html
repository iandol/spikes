Spike Train Analysis Toolkit
Example datasets and toolkit demos

Data files

In the data/ directory, there are five data sets:
synth
Contains data like that described in Victor & Purpura (1997), Figure 5(E-H).
taste
Contains the data from the experiment described in Di Lorenzo and Victor (2003). Specifically, it contains the data from "cell 11", which is analyzed in Figure 6.
drift
Contains the data from the experiment described in Reich et al. (2001). Specifically, it contains the drifting grating responses from cell 44/9s, which are depicted in the lower right of Figure 2.
phase
Contains the data from the experiment described in Aronov et al. (2003). Specifically, it contains the responses from cells 410106.s and 410106.t, which are depicted in Figure 1B. Because the full dataset depicted in the paper is large, only data corresponding to every fourth phase is included.
inforate_uni and inforate_rep
These sets consist of synthetic data inspired by the experiment described in Reinagel and Reid (2000). inforate_uni consists of unique Poisson spike trains with a rate of 40 spikes/s. inforate_rep consists of repeated spike trains, where each spike in the set is jittered according to a Gaussian distribution with a standard deviation of 0.6 ms.

Demo scripts

The example m-file scripts can be found in the demo/ directory. Links to images of the output figures are provided below so that you can verify that the toolkit is functioning properly. The performance of entropy and information methods is data-dependent. The results depicted in these demos are specific to the datasets analyzed—no conclusions about the methods' performance on other datasets are implied or should be inferred.

Entropy method demo

The script demo_entropy demonstrates all of the entropy methods included in the toolkit on examples drawn from a binomial distribution with 10 bins. View the expected results. Dataset generation and bootstrap variance estimation make use of a pseudorandom number generation. Therefore, results may differ slightly from system to system.

Formal information demo

The script demo_directformal demostrates the use of the directformal function to compute the formal information rate on the inforate_uni and inforate_rep dataset. View the expected results.

Category-specific information demos

There are three scripts that demonstrate how the toolkit quantifies the amount of information that an ensemble of spike trains conveys about a stimulus that can be characterized by discrete categories: demo_directcat, demo_metric, and demo_binless. When run, these scripts asks you to choose the synth, taste or drift datasets. To bypass this query, you may set the dataset variable (e.g., dataset='synth') before running the scripts. (Note that you must clear this variable in order to receive the query again.) All of these demos can be run in succession with phase and taste datasets using the demo_all script. Analysis of the drift dataset can take a long time on some computers, and therefore is not included in the demo_all script.

The script demo_metric_multi demonstrates the use of the metric space method on the multineuron dataset phase. View the expected results. This demo is very memory intensive. If you have trouble running it, try reducing the size of the analysis window by changing the values of opts.start_time and opts.end_time.
